# Branches

## Context
- Branch names follow `<type>/<kebab-case-slug>`.
- Common types: `feat`, `fix`, `refactor`, `chore`, `docs`.

## DOs
- Pick the type that matches the *intent* of the work, not the largest file you'll touch.
- Keep the slug short — 3–5 words after the type prefix is plenty.
- Lowercase, alphanumerics + hyphens only.

## DONTs
- No `<initials>/...` prefixes; the type prefix is the only one we use.
- No issue numbers in the slug — link them in the PR description instead.
